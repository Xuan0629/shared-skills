# P4: Shared-Skills Web UI Dashboard

## Goal
Build a lightweight web dashboard at `~/shared-skills/webui/` for managing the shared-skills system. The dashboard is read-only (write ops stay in CLI). Pure static HTML + CSS + JS — zero framework dependencies.

## Data Source
Create `webui/data.py` that scans the filesystem and outputs `webui/data.json`:
- List all agents found on the system (hermes, claude, codex, openclaw)
- List all skills in shared pool with their sync_to targets
- For each skill-agent pair: status (synced/outdated/missing/disabled)
- Count shared skills per agent

## Dashboard Features

### 1. Agent Overview
- Cards for each agent showing: name, skill count, last sync time
- Color coded: green (all synced), yellow (some outdated), red (none synced)

### 2. Skill Table (main view)
- Columns: skill name, description, synced agents (icon per agent), status, last updated
- Search/filter by skill name
- Filter by status: all / synced / outdated / disabled
- Filter by agent: "shared with 2+ agents", "shared with Claude only", etc.
- Click a skill → expand to show per-agent details (hash, path, last sync)

### 3. Quick Actions (links to CLI, not buttons that execute)
- "Sync all" → shows the CLI command: `python3 sync-skills.py`
- "Check status" → shows: `python3 sync-skills.py --check`
- "View log" → shows: `python3 sync-skills.py --show-log`

## Tech Constraints
- Pure HTML + CSS + JS. No React, no npm, no build step.
- Dark theme matching Unison dashboard style
- Data loaded from `data.json` (generated by `python3 data.py`)
- Mobile responsive
- Bilingual (EN/CN toggle)

## Files to Create
- `webui/index.html` — main page
- `webui/style.css` — styling
- `webui/app.js` — logic
- `webui/data.py` — data generator (run `python3 data.py` to refresh data.json)
- `webui/data.json` — generated, gitignored

## Verify
Run `python3 webui/data.py && python3 -m http.server -d webui 8080` and open in browser.
